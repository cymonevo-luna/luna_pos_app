## Manual device QA procedure (for human tester)

### Setup (both scenarios)

1. Pair Bluetooth thermal printer in Android system settings.
2. Install release APK built with `API_BASE_URL=https://pos-api.cymonevo.com`.
3. Log in as `cashier-test@cymonevo.com` / `LunaTesting123!`.
4. Settings → Printer: select paired printer, tap **Test Print** — expect ESC/POS test receipt.
5. Confirm POS home shows **Es Teh** and **Nasi Goreng** with stock > 0.

### Scenario A — Checkout prints receipt

1. Add Es Teh to cart.
2. Checkout with cash, **Print receipt** enabled.
3. Complete payment.

**Expected:** Transaction succeeds, cart clears, physical receipt prints with line items, totals, and store header (Luna Coffee / Cabang Sudirman).

### Scenario B — Retry after transient disconnect

1. Fresh cart, printer connected, **Print receipt** enabled.
2. During print attempt, power-cycle printer or briefly disable Bluetooth, then restore.
3. Complete payment.

**Expected:** Sale completes (`transactionId` returned, cart cleared) even if initial print fails. UI shows specific `BluetoothPrinterException` / `PrinterMessages` error with retry action.

4. Tap **Retry print** with printer back online.

**Expected:** Receipt prints without duplicate transaction or re-checkout.

### Failure capture checklist

If any scenario fails on device:

- `adb logcat | grep -iE 'Bluetooth|print|ReceiptPrint'`
- Screenshot of error snackbar
- Steps to reproduce
- File new BUG in luna_pos_app — do not mark POS-97-3 complete
